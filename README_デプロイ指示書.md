# 坊っちゃん レシピアプリ — デプロイ指示書（Claude Code 用）

## これは何
焼き鳥おでん坊っちゃんの、現場スタッフ向けレシピ参照アプリ（YouTube型UI）。
レシピDB SS / 商品マスタSS から出力したJSONを読んで表示する静的Webアプリ。

## ファイル構成（このリポジトリに置くもの）
```
/
├ index.html      … アプリ本体（単一HTML。CSS/JS/フォント/日本語分割エンジン同梱）
├ recipes.json    … レシピデータ（レシピDB SS → JSON出力GASで生成）
└ orders.json     … 発注データ（商品マスタSS → 発注JSON出力GASで生成）
```
- index.html は起動時に `./recipes.json` と `./orders.json` を fetch で読む
- 3ファイルを同じ階層に置くだけで動く（ビルド不要・依存パッケージなし）

## やってほしいこと
1. この3ファイルでGitHubリポジトリを作る（例: `bocchan-recipe`）
2. Vercel に連携してデプロイ（フレームワークなし＝「Other / 静的」でOK）
3. 公開URLを発行（現場がスマホで開く）

## デプロイ設定メモ（Vercel）
- Framework Preset: **Other**（静的HTML）
- Build Command: なし（空）
- Output Directory: ルート（`.`）
- index.html がそのまま配信されればOK

## 運用フロー（デプロイ後のデータ更新）
```
① レシピDB SS を編集
② SSメニュー「レシピDB管理 → JSONを出力」→ ドライブに recipes.json
③ 商品マスタSS メニュー「IM整理 → 発注JSONを出力」→ ドライブに orders.json
④ ②③のJSONをこのリポジトリに上書きして git push
⑤ Vercel が自動で再デプロイ → アプリ最新化
```
※ ④の「ドライブ→GitHub」は当面手動。将来 GAS から GitHub API で直接push する自動化も可能（トークン設定が必要）。

## 技術メモ
- 日本語あいまい検索：Fuse.js（CDN）＋ TinySegmenter（HTML内に同梱）
- 画像/動画：Google DriveのファイルIDから thumbnail/preview を読む。共有設定「リンクを知っている全員」が前提（アカウント無しで見れる必要あり）。坊っちゃんは現状メディア未登録のため絵文字プレースホルダ表示。
- 発注先：店舗別ロジックの「器」はアプリ側に用意済み。現状はラベル→業者1対1（坊っちゃん）。将来かっちゃんで店舗別になる場合は stores.json（店舗→業者）を追加し、店舗セレクタを有効化する。

## 注意
- ブラウザのlocalStorage等は使っていない（静的配信のみ）
- recipes.json / orders.json は UTF-8。文字化けする場合はエンコーディング確認
- レシピの掲載/非掲載はSS側の「掲載」列で制御済み（JSON出力時にFALSEは除外される）
